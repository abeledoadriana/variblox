Operadores

¿Qué son los operadores?

Los operadores son símbolos especiales que le dicen a JavaScript que realice alguna operación con uno o más valores (llamados operandos).

Los operadores son símbolos o palabras clave que se utilizan para realizar operaciones sobre uno o más valores (o variables), produciendo un nuevo valor.

Permiten realizar cálculos matemáticos, comparaciones lógicas, asignaciones y muchas otras acciones.

Tipos de operadores

Operadores aritméticos: Para realizar cálculos matemáticos. Ejemplo: 

let a = 15;

let b = 12;

console.log(a + b); // Resultado: 27

console.log(a - b); // Resta: 3

console.log(a * b); // Multiplicación: 180

console.log(a / b); // División: 1.25

console.log(a % b); // Módulo (resto de la división): 3

 

Operadores de asignación

Se usan para asignar un valor a una variable. El más común es =, pero hay 0tros. Por ejemplo: 

let puntaje = 100; // Asigna 100 a puntaje
puntaje += 50; // Es lo mismo que: puntaje= puntaje+ 50; (ahora es 150)
puntaje -= 20; // Es lo mismo que: puntaje= puntaje- 20; (ahora es 130)

Operadores lógicos
Se usan para combinar o negar condiciones booleanas.

let llueve = true;
let nublado= false;

// AND (&&): Verdadero solo si ambas condiciones son verdaderas
console.log(llueve&& nublado); // false (porque nublado es false)

// OR (||): Verdadero si al menos una condición es verdadera
console.log(llueve || nublado); // true (porque llueve es true)

// NOT (!): Invierte el valor booleano
console.log(!llueve); // false

Operadores de comparación

Se utilizan para comparar dos valores.

El resultado de una comparación siempre es un valor booleano: true o false.

(Abarcan todos los operadores que comparan valores, no importa si son de igualdad o de relación).
Ejemplo: ==, ===, !=, !==, >, <, >=, <=.
Los operadores relacionales son un subconjunto de los de comparación, usados específicamente para relaciones de orden (mayor, menor, mayor o igual, menor o igual).
Ejemplo: >, <, >=, <=.

Ejemplo:

Se recomienda usar === y !== en lugar de == y != para evitar errores por conversión automática de tipos (coerción).
Por ejemplo: 0 == false es true, pero 0 === false es false.
